# 开源绩效考核系统调研报告

*生成日期: 2026-06-11 | 调研来源: GitHub、官方网站、技术文档 | 系统数量: 7*

---

## 一、调研背景与需求

### 核心功能需求
1. **月度任务推送与自评**：月底推送领导确认完的任务给员工，员工进行自评
2. **领导打分**：自评完成后，领导进行评分
3. **数据采集**：采集员工考勤、各系统流程审判人员的效率考评等信息
4. **互评功能**：随机选择10人进行互评
5. **AI/大模型打分**：调用大模型进行多维度自定义规则打分

### 1. ERPNext HR (Frappe HRMS) ⭐⭐⭐⭐⭐

**基本信息**
- **GitHub**: https://github.com/frappe/hrms
- **Stars**: 8.1k
- **技术栈**: Python, JavaScript, Vue.js
- **开源协议**: GPL-3.0
- **官网**: https://frappe.io/hr

**核心功能**
- ✅ **绩效管理模块**：完整的 Performance Management 功能
- ✅ **目标管理**：支持 KRA (Key Result Areas) 和 KR (Key Results) 对齐
- ✅ **自评功能**：员工自我评估与打分
- ✅ **评估周期**：支持自定义评估周期管理
- ✅ **自定义评分**：支持自定义公式和加权评分
- ✅ **考勤集成**：内置考勤管理模块
- ⚠️ **互评功能**：需二次开发或通过自定义表单实现
- ⚠️ **AI打分**：需集成外部LLM API

**优势**
- 功能完整，社区活跃（8.1k stars）
- 内置考勤、薪资等完整HR模块
- 支持自定义评分公式
- 文档完善，部署简单

**劣势**
- 互评功能需二次开发
- AI打分需自行集成

## 五、综合推荐

### 🥇 推荐方案1：ERPNext HR (Frappe HRMS)

**推荐理由**
- **功能完整度最高**：原生支持自评、经理评估、目标管理、考勤集成
- **社区活跃**：8.1k stars，文档完善
- **技术栈现代**：Python + Vue.js，易于扩展
- **自定义能力强**：支持自定义评分公式、评估周期
- **部署简单**：相比 Odoo 更轻量

**需二次开发的部分**
- 互评功能（可通过自定义表单或插件实现）
- AI 打分（需集成 LLM API，但框架支持良好）

**实施建议**
1. 部署 ERPNext HRMS 基础环境
2. 配置绩效管理模块（KRA、评估周期）
3. 开发互评插件（随机选择评估人）
4. 集成 LLM API（通过 Server Script 或自定义 App）
5. 配置考勤数据同步

**推荐理由**
- **360° 评估原生支持**：最符合"自评+领导评+互评"需求

## 七、总结

### 最佳选择

**对于您的需求（自评+领导评+互评+AI打分+考勤集成）：**

1. **首选 ERPNext HR**：功能完整度最高，社区活跃，自定义能力强
2. **备选 Odoo Appraisal**：360° 评估原生支持，但系统较重
3. **创新方案 NocoBase**：低代码快速搭建，AI 原生集成

### 关键决策点

- **如果追求快速上线**：选择 ERPNext HR，二次开发量最小
- **如果追求功能完整**：选择 Odoo Appraisal，360° 评估最完善
- **如果追求高度定制**：选择 NocoBase，低代码灵活搭建
- **如果预算有限**：选择 Sentrifugo 或 IceHrm，轻量且完全开源

### 风险提示

1. **互评功能**：大部分系统需要二次开发，建议提前规划
2. **AI 打分**：所有系统都需要集成外部 LLM API
3. **数据采集**：需要与现有业务系统集成，工作量取决于系统复杂度
4. **维护成本**：Odoo 等重型系统维护成本较高
