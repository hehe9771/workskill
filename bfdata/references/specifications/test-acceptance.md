# bfdata 技能测试验收标准

## 测试环境要求

| 项目 | 要求 |
|------|------|
| Python | 3.10+ |
| 压缩工具 | 7-Zip 或 tar（至少其一） |
| 测试目录 | 至少一个可访问的测试目录 |

## 验收测试用例

### TC-001: 环境初始化

**目标**: 验证 `init-env.sh` 正确生成配置文件

**步骤**:
1. 删除 `assets/static/.env`（如存在）
2. 运行 `bash scripts/setup/init-env.sh`
3. 检查 `assets/static/.env` 是否存在
4. 检查 `assets/configs/backup-config.json` 是否存在

**通过标准**:
- `.env` 文件从模板成功生成
- 配置文件从模板成功生成
- 重复运行不覆盖已有文件

### TC-002: 单目录备份

**目标**: 验证单个目录备份成功

**步骤**:
1. 设置 `BFDATA_SOURCES=["<测试目录路径>"]`
2. 运行 `python scripts/data-processing/bfdata.py`
3. 检查输出目录是否存在归档文件

**通过标准**:
- 脚本退出码为 0
- 归档文件存在且大小 > 0
- 归档包含测试目录内容

### TC-003: 多目录备份

**目标**: 验证多个目录合并到单个归档

**步骤**:
1. 设置 `BFDATA_SOURCES=["<目录1>","<目录2>","<目录3>"]`
2. 运行备份脚本
3. 验证归档包含所有顶层目录

**通过标准**:
- 归档文件包含所有指定目录
- 不存在的路径被跳过（警告但不报错）

### TC-004: 不存在路径处理

**目标**: 验证不存在的路径被正确处理

**步骤**:
1. 设置 `BFDATA_SOURCES` 包含一个不存在的路径
2. 运行备份脚本

**通过标准**:
- 输出 `[WARN]` 警告信息
- 不阻断备份流程
- 其他有效路径正常备份

### TC-005: 压缩工具自动检测

**目标**: 验证压缩工具优先级 7z > tar > 报错

**步骤**:
1. 在不安装 7z 的环境中运行（使用 tar）
2. 设置 `BFDATA_COMPRESS_CMD=7z` 但在无 7z 环境中运行
3. 两者都不可用时运行

**通过标准**:
- 有 7z 时使用 7z
- 无 7z 有 tar 时使用 tar
- 两者都无时输出 `[ERROR]` 并退出

### TC-006: 环境变量优先级

**目标**: 验证环境变量正确覆盖配置

**步骤**:
1. 在 `.env` 中设置源路径 A
2. 同时通过命令行环境变量设置源路径 B
3. 验证两者都被包含

**通过标准**:
- 配置文件的 `sources` 和环境变量的 `BFDATA_SOURCES` 合并
- 环境变量值不覆盖 `.env` 值，而是追加

### TC-007: 路径遍历防护

**目标**: 验证路径遍历攻击被拒绝

**步骤**:
1. 设置 `BFDATA_SOURCES` 包含 `../etc/passwd` 类路径
2. 运行备份脚本

**通过标准**:
- 输出 `[ERROR] 拒绝路径遍历尝试`
- 脚本退出码非 0

### TC-008: OSS 上传（有凭证）

**目标**: 验证 OSS 上传功能

**前提**: 已配置有效的 OSS 凭证

**步骤**:
1. 配置 `.env` 中的 OSS 变量
2. 运行备份脚本
3. 检查 OSS 中是否存在对应文件

**通过标准**:
- 输出 `[OK] OSS 上传成功`
- OSS 中可找到对应文件

### TC-009: OSS 上传失败回退

**目标**: 验证 OSS 上传失败时本地归档保留

**步骤**:
1. 配置错误的 OSS 凭证
2. 运行备份脚本

**通过标准**:
- 输出 `[WARN]` 或 `[ERROR]` 上传失败信息
- 本地归档文件保留
- 脚本退出码为 0（备份成功，仅上传失败）

### TC-010: 归档验证

**目标**: 验证 `validate-backup.sh` 正确校验归档

**步骤**:
1. 对有效归档运行验证脚本
2. 对不存在的文件运行验证
3. 对损坏的文件运行验证

**通过标准**:
- 有效归档: 输出 `[OK]`，退出码 0
- 不存在文件: 输出 `[FAIL]`，退出码非 0
- 损坏文件: 输出 `[FAIL] 归档损坏`，退出码非 0

### TC-011: 零硬编码验证

**目标**: 验证脚本中无硬编码路径或凭证

**步骤**:
1. 搜索脚本中的绝对路径
2. 搜索脚本中的密钥/密码

**通过标准**:
- 除模板和注释外，无硬编码路径
- 无硬编码凭证

### TC-012: 跨平台路径处理

**目标**: 验证 Windows/macOS/Linux 路径兼容

**步骤**:
1. Windows: 使用 `D:/path` 格式路径
2. 验证环境变量展开（`%USERPROFILE%` 或 `$HOME`）

**通过标准**:
- Windows 路径正确展开
- POSIX 路径正确展开
- 归档内容在对应平台可正常解压

## 迭代修复策略

如果任何测试用例失败：

1. **记录失败详情**: 输出错误日志、环境信息、复现步骤
2. **根因分析**: 确定是代码 bug、环境问题还是测试用例问题
3. **修复**: 最小修改原则，只修复必要的部分
4. **回归测试**: 修复后重新运行所有测试用例
5. **重复直到全部通过**

## 测试执行命令

```bash
# 设置 Python 路径
export BFDATA_PYTHON="C:/Users/wuyan/.conda/envs/picproject/python.exe"

# 1. 初始化环境
bash bfdata/scripts/setup/init-env.sh

# 2. 编辑 .env 配置备份源
#    编辑 bfdata/assets/static/.env

# 3. 执行备份
$BFDATA_PYTHON bfdata/scripts/data-processing/bfdata.py

# 4. 验证归档
bash bfdata/scripts/validation/validate-backup.sh
```
