# 视频处理标准规范

## 代码标准

### 脚本编写标准
- 所有脚本使用bash编写
- 使用`#!/bin/bash`作为shebang
- 设置`set -e`以确保错误中断
- 所有函数应有适当的错误处理

### 变量命名约定
- 使用大写字母和下划线：`VIDEO_FILE`
- 局部变量使用小写：`temp_file`
- 避免使用容易混淆的字符

## 输出标准

### Markdown格式标准
- 标题使用ATX风格：`# 主标题`
- 列表使用连字符：`- 项目`
- 代码块使用三个反引号：```mermaid

### 文本处理标准
- 使用UTF-8编码
- 使用Unix换行符(\n)
- 行长度不超过120字符

## 性能标准

### 处理速度
- 10分钟视频应在5分钟内处理完成
- 处理速度不应低于1:1比例
- 大于1小时的视频需要分段处理

### 资源使用
- 内存使用不超过系统内存的50%
- CPU使用率平均不超过75%
- 临时文件大小不超过原视频的2倍

## 错误处理标准

### 错误消息格式
- 错误消息应以"错误:"开头
- 包含具体的文件名或操作名
- 提供解决建议

### 日志记录
- 记录所有重要操作的时间戳
- 包含处理的输入和输出文件
- 错误日志包含堆栈跟踪信息

## 安全标准

### 文件访问
- 验证所有输入路径以避免路径遍历
- 使用绝对路径而不是相对路径
- 临时文件应设置适当的权限

### API调用
- 所有API调用必须有超时设置
- 验证API响应的有效性
- 避免在日志中记录敏感信息

## 验证标准

### 输入验证
- 验证视频URL格式
- 检查文件大小是否超出限制
- 验证视频格式是否支持

### 输出验证
- 验证生成的Markdown语法有效性
- 检查所有图表是否正确嵌入
- 确认章节结构完整性
