# 视频处理技术规范

## 概述

本文档定义了videobook技能中视频处理的技术规范和标准，包括支持的视频格式、处理参数、性能指标等。

## 支持的视频平台

### 主要支持平台
- YouTube
- Bilibili
- Vimeo
- TikTok
- Twitter/X
- Instagram

### 视频格式支持

| 格式 | 支持级别 | 备注 |
|------|----------|------|
| MP4 | 首选 | H.264编码，最高支持4K分辨率 |
| AVI | 标准 | 基础支持，部分编解码器可能不兼容 |
| MOV | 标准 | Apple QuickTime格式 |
| WMV | 有限 | 需要额外编解码器支持 |
| FLV | 有限 | Flash视频格式 |
| WEBM | 标准 | Google开发的开放格式 |

## 音频提取规范

### 参数设置
- 采样率：16kHz（Whisper API推荐）
- 位深度：16-bit
- 声道：Mono（单声道）

### 输出格式
- 默认：WAV格式（适用于Whisper API）
- 备用：MP3格式（高压缩比）

### 音频质量保证
- 信噪比 ≥ 20dB
- 音频清晰度检测
- 去除静音片段（< 0.1秒）

## 文本转换规范

### 分段规则
- 按时间戳分段（每分钟一段）
- 按语义分段（保持句子完整性）
- 最小段长度：50词
- 最大片段长度：500词

### 时间戳格式
- 标准格式：HH:MM:SS
- 示例：00:01:30 表示1分30秒

### 文本预处理
- 去除填充词汇（嗯、啊等）
- 标准化标点符号
- 修正语音识别错误

## 处理性能指标

### 处理时间
- 10分钟以内视频：≤ 5分钟处理时间
- 10-30分钟视频：≤ 15分钟处理时间
- 30-60分钟视频：≤ 30分钟处理时间

### 资源消耗
- CPU使用率：峰值 ≤ 80%
- 内存使用：≤ 2GB RAM
- 存储空间：临时文件不超过视频大小的3倍

## 错误处理规范

### 重试机制
- 网络错误：最多重试3次，每次间隔递增
- API错误：按API提供方建议的重试策略
- 磁盘空间不足：立即停止并提示用户

### 降级策略
- 视频下载失败：尝试备用下载源
- 音频提取失败：尝试不同的音频提取工具
- API调用失败：使用离线模型或模拟输出

## 兼容性要求

### 系统要求
- 操作系统：Linux/macOS/Windows 10+
- 内存：≥ 4GB RAM（推荐8GB+）
- 存储：≥ 1GB可用空间（取决于视频大小）

### 依赖版本
- opencli：≥ 1.0.0
- ffmpeg：≥ 4.0
- Python：≥ 3.7
- Node.js：≥ 14.0
